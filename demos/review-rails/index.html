The five deterministic rule types

What runs before any model does

forbidden-pattern

Regex against added lines, optionally scoped by path glob. Catches things like stray console.log.

required-pairing

If a file matching one glob changes, a file matching another must change too — e.g. src changes require test changes.

size-budget

Caps lines-per-file and/or files-per-PR, with label-based overrides for legitimate large changes.

path-policy

Restricts sensitive paths to allowed authors/teams; denies symlink tricks into protected paths by default.

commit-format

Enforces a commit-message regex, e.g. Conventional Commits.

judgment (LLM-only)

No deterministic backing — a rubric routed to the LLM pass, budgeted and schema-validated, dropped entirely rather than posted if it fails validation twice.
